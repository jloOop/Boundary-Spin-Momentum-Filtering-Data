# Representative S4 time-window diagnostics

This folder contains compact JSON audit files for the Supplement S4 time-window diagnostics.

The CSV files in `data/processed-summary-tables/` are the main public data products. They summarize the cutoff sweep, roof detector budget, first-bounce plane diagnostics, and no-return/post-cut current ratios.

The JSON files here provide the provenance behind those tables.

- `metrics/`: compact diagnostic summaries for each value of $\omega$. These are the recommended files for reproducing the public CSV tables.
- `full-summary/`: lower-level solver summaries with detailed window choices, thresholds, search intervals, and peak-finding diagnostics. These are included only as representative audit files.

The detector observable is always the roof flux / norm loss. Interior-plane timing windows and $J_-/J_+$ ratios are diagnostic tools used to estimate first-pass axial motion and separate early first-pass behavior from later finite-guide return structure. They are not additional detectors and not a separate arrival-time law.


# Representative S7 boundary-layer JSON summaries

This folder contains compact sanitized JSON audit files generated by
`diagnose_boundary_symbol.py`.

The public CSV tables in `data/processed-summary-tables/` are the main data products.
The JSON files here provide provenance for those tables and preserve the deeper diagnostic
structure without committing large raw `.npy` arrays.

## Files

```text
s7_boundary_symbol_summary_omega001.json
s7_boundary_symbol_summary_omega050.json
s7_boundary_symbol_summary_omega100.json
s7_boundary_symbol_summary_omega200.json
s7_boundary_symbol_summary_omega300.json
```

The set matches the S7-style diagnostic sweep used for the boundary-symbol mechanism.
No `omega=150` S7 JSON was included in this upload. That is acceptable for the S7
diagnostic table; if an `omega=150` file is produced later, rerun
`tools/build_s7_boundary_layer_tables.py` and the tables will include it.

## What is inside each JSON?

Each JSON includes:

- run/grid metadata,
- top-row detector-budget checks,
- one-grid boundary-symbol diagnostics,
- finite-epsilon exact/taylor checks,
- covariance coefficient data,
- two-branch Duhamel/slab depth diagnostics,
- flags confirming that `Pi_-` is not neglected.

Absolute HPC scratch paths were replaced with `not_public`; a clean `run_id` field is kept.
